1. データアクセスGUI

1.1 概要

データアクセスGUIは，StkData利用プログラムの１つで，StkDataが管理するデータに対する操作をGUIで提供します。

1.2 要件

データアクセスGUIは以下の環境で動作します。
  • 動作に必要なCPU:
    1GHz以上のクロック周波数で動作するCPU

  • 動作に必要なメモリ容量:
    512MBytes以上

  • 導入に必要な空きディスク容量:
    10MBytes以上

1.3 画面仕様

データアクセスGUIのウィンドウは選択したテーブルのレコード一覧を表示します。 フィルタリング条件が指定してある場合は，絞り込んだ結果をレコード一覧に表示します。

データアクセスGUIが提供する機能を下記に示します。

[File]メニューに関連する機能

(1) Save As
StkDataが管理するデータ（テーブル情報／レコード情報）をファイルに書き込みます。
ファイル名はダイアログで指定できます。
(2) Save
StkDataが管理するデータ（テーブル情報／レコード情報）を現在編集中のファイルに書き込みます。
(3) Open
ファイルからデータを読込みます。
(4) Exit
データアクセスGUIを終了します。

[Table]メニューに関連する機能

テーブルを選択します。テーブルが1つも存在しない場合，[テーブル]メニュー以下にテーブル一覧が表示されません。

[Operation]メニューに関連する機能

(1) Filtering
指定したフィルタリング条件で表示するレコードを絞り込みます。
最大で5つの条件を同時に指定することができます。
(2) Insert
レコードを追加します。
バイナリ型カラムを除く全てのカラムに値を入力する必要があります（バイナリ型カラムには 値を入力することができません）。 文字列型カラム，ワイド文字列型カラムに空文字を指定することはできません。特に値が決まっていない 文字列型カラム，ワイド文字列型カラムにはダミー文字列（例：空白文字）を指定してください。
同様に整数型カラムに空文字列を指定することはできません。特に値が決まっていない 整数型カラムにはダミー値（例：0）を指定してください。
(3) Update
選択したレコードを更新します。
最大で5つのカラムの値を同時に変更することができます。
(4) Delete
選択したレコードを削除します。
(5) Create Table
テーブルを新規に作成します。
(6) Delete Table
現在選択されているテーブルを削除します。
(7) Show Table Information
現在選択されているテーブルの情報を表示します。
(8) Refresh
レコード一覧をリフレッシュします。

その他の機能
(1) レコードのソート
ウィンドウ上方のカラム名を押すと，そのカラムの値で昇順となるようにレコードをソートします。 同じカラム名を再度押すと，降順でソートします。
